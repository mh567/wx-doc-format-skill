# WX 文档格式转换

将 Markdown 或 DOCX 文档转换为模板驱动的 WX 格式 `.docx` 文件。

## 核心功能

- 基于 Word OOXML 编号定义识别列表，保留源编号证据、语义层级和分组重启
- 自动识别 Word 目录、TOC 样式目录和手工目录，移除重复源目录后生成 WX 标准目录
- 自动识别并移除源封面，提取唯一文档标题，固定输出为“目录、文档标题、正文”
- 将单单元格自然语言说明框识别为 `callout`，与接口代码示例和布局容器统一禁止自动题注
- 图片、表格完整保留，自动编号，输出样式与模板完全一致
- 表格采用幂等格式收口，保留 0.69 cm 最小行高和 `atLeast`，清除会覆盖“表正文”的源段落间距与行距
- 可选 LLM 增强：模糊目录候选复核、功能点列表识别、表格题注自动生成

## 一键安装

对任何支持 skill 的 Agent 说：

> 从 GitHub 仓库 mh567/wx-doc-format-skill 安装 wx-doc-format skill

更新时重复上述指令即可。

## 使用方法

触发词：**wx文档格式**

发送文档后 Agent 会自动提供两个模式选择：
1. 普通模式：纯规则转换
2. LLM 增强模式：在普通模式能力上增加模糊目录复核、功能点列表识别和题注生成

两种模式都会运行确定性目录检测和 Word 编号预分析。普通模式根据编号定义、列表样式、连续编号组和保护角色自动转换高置信度列表；孤立的普通样式编号段落保留为正文并记录风险。源文档的 `numFmt` 和 `lvlText` 用作识别证据，WX 输出样式按照 AST 层级统一选择：一级有序使用一级字母列项，二级及以下有序使用二级数字列项；一级无序使用一级横线列项，二级及以下无序使用二级项目符号。LLM 增强模式通过注册式 `toc_region_review` 复核模糊目录候选，并通过 `list_detect` 复核模糊编号候选和语义列表。

两种模式也会执行确定性首部规范化。源封面整体从输出排除，主标题按源文本、Word 核心属性和文件名的顺序降级提取。输出的第一部分始终是根据 Heading 样式生成的唯一主目录，分页后紧接唯一文档标题和正文。

表格语义采用确定性分类：具有行列关系的表格为 `data`，单单元格接口报文为 `code_sample`，单单元格自然语言说明框为 `callout`，纯图片或嵌套内容容器为 `layout`。只有 `data` 表允许自动生成题注；源文档已有的人工题注继续保留。普通模式和增强模式共享相同分类结果，`caption_gen` 只接收允许自动题注的数据表。

转换报告包含源列表数量、AST 列表数量、输出列表数量、样式层级一致性、编号重启组和表格语义审计，可用于检查列表是否完整保留、WX 层级样式是否正确，以及非数据表是否带有错误的自动题注。

表格格式由统一策略控制：模板表格样式负责表级外观，0.69 cm 与 `atLeast` 负责行级下限，“表正文”负责段落排版，代码示例通过显式角色保持左对齐。输出审计会检查表格样式 ID、行高、单元格边距、对齐和段落直接格式残留。重复执行规范化不会继续改变表格 OOXML。

详细架构见 `SKILL.md`。
